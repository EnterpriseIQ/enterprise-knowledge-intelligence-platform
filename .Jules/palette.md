## 2024-03-24 - FAQ Accordion Accessibility Enhancement
**Learning:** Custom disclosure/accordion components built using generic structural elements (e.g., `div` wrapping a `<button>`) combined with Framer Motion frequently omit standard accordion accessibility linkage. Missing `aria-expanded` and the `aria-controls` / `aria-labelledby` pair leaves screen readers unnotified when sections expand or collapse. Also, decorative icons (`ChevronDown`) inside such buttons lack `aria-hidden="true"`, adding clutter to screen readers.
**Action:** When auditing custom UI components with expandable sections, explicitly verify that the toggling button maintains the correct `aria-expanded` state, points to the content via `aria-controls`, and that the content region has `role="region"` with an `aria-labelledby` attribute referring back to the button. Ensure icons are properly hidden.
